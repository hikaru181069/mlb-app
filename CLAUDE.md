CLAUDE.md

Project

このプロジェクトは学習用・ポートフォリオ用のMLBアプリです。

プロダクトの目的や方向性は以下を参照してください。

- docs/vision.md
- docs/roadmap.md

方向性が変わった場合は、CLAUDE.mdではなく vision.md を更新してください。

⸻

Development Principles

以下を優先してください。

- 可読性
- 保守性
- 理解しやすさ
- 小さな変更
- 既存機能を壊さないこと

過度な抽象化や過剰設計は避けてください。

⸻

Before Large Changes

大規模な変更を行う前に説明してください。

- 問題点
- 改善案
- メリット
- トレードオフ

説明なしで大規模な変更を行わないでください。

⸻

Code Organization

1つのファイルにロジックを集中させないでください。

責務ごとに分割してください。

例：

- routers
- services
- models
- utils

⸻

Explanation Style

説明するときは以下を重視してください。

- 初学者向け
- なぜその設計なのか
- データの流れ
- 各ファイルの役割
- 面接で説明できるレベル

専門用語はできるだけ分かりやすく説明してください。

⸻

Learning Priority

高度な設計よりも理解しやすさを優先してください。

今の学習段階で不要な高度な概念は無理に導入しないでください。
